How to add a new A/B test variant

Option A — Separate URLs (recommended for Meta ads):

  1. Duplicate the landing page folder in the GitHub repo (e.g. bitebot-vsl-v2/)
  2. Change the headline or whatever you're testing
  3. In the tracking script at the bottom, change variant_name from "control" to your test name (e.g. "new-headline")
  4. Ask Claude to register the new variant in Supabase
  5. Point one ad to /bitebot-vsl-landing/ and another to /bitebot-vsl-v2/

Option B — Single URL with auto-split:

  1. Use the /assign endpoint to randomly assign visitors to variants
  2. The page loads, calls /assign?page_slug=bitebot-vsl-landing, gets back a variant name
  3. JS swaps the headline based on the assigned variant
  4. Same URL, 50/50 split, tracked automatically
